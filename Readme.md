# Teledoc API

Teledoc API — backend-приложение для работы с клиентами.

Проект реализован в рамках тестового задания «Теледок».  
Основная задача — создать API для просмотра, добавления, редактирования и удаления клиентов.

Клиентами в системе выступают:

- юридические лица
- индивидуальные предприниматели

Юридические лица могут иметь учредителей.  
Индивидуальный предприниматель может иметь только одного учредителя.

В проекте реализована валидация ИНН для юридических лиц и физических лиц / индивидуальных предпринимателей.  
Проверка ИНН выполняется по длине, цифровому формату и контрольным суммам.

Для хранения данных используется PostgreSQL.  
Работа с базой данных реализована через Entity Framework Core Code First.

Проект запускается через .NET Aspire, который используется как оркестратор приложения и инфраструктурных зависимостей.

## Требования для запуска

Для запуска проекта требуется:

- .NET Aspire 13.2.4
- Visual Studio 2026
- .NET 10 SDK
- Docker Desktop

.NET Aspire используется как оркестратор приложения.

Docker Desktop требуется для запуска PostgreSQL в контейнере.

PostgreSQL используется как основная база данных приложения.


## Что реализовано

В проекте реализованы основные слои для работы с клиентами.

### Репозиторий клиентов

Репозиторий отвечает за работу с базой данных через Entity Framework Core.

Реализованные операции:

- получение клиента по ID
- получение клиента по ИНН
- получение списка всех клиентов
- добавление клиента
- обновление клиента
- удаление клиента
### Сервис управления клиентами

Сервис содержит основную бизнес-логику приложения.

Через сервис выполняются:

- получение всех клиентов
- получение клиента по ID
- получение клиента по ИНН
- создание клиента
- обновление клиента
- удаление клиента

### Валидация ИНН

В проекте реализована валидация ИНН через отдельный валидатор и атрибут `InnAttribute`.

Атрибут используется в DTO-моделях запросов и позволяет валидировать ИНН на уровне входных данных API.

Проверяется:
- наличие значения
- корректный формат
- контрольные суммы ИНН

### Контроллер клиентов

Контроллер предоставляет REST API для работы с клиентами.

Базовый маршрут контроллера:

```http
/api/v1/clients
```

Реализованные endpoints:

| Метод  | URL                            | Описание                |
| ------ | ------------------------------ | ----------------------- |
| GET    | `/api/v1/clients`              | Получить всех клиентов  |
| GET    | `/api/v1/clients/{id}`         | Получить клиента по ID  |
| GET    | `/api/v1/clients/by-inn/{inn}` | Получить клиента по ИНН |
| POST   | `/api/v1/clients`              | Создать клиента         |
| PUT    | `/api/v1/clients/{id}`         | Обновить клиента        |
| DELETE | `/api/v1/clients/{id}`         | Удалить клиента         |

### Swagger

В проект подключен Swagger / OpenAPI.

Swagger UI доступен по пути:

```text
/swagger
```

Настройка Swagger endpoint:

```csharp
options.SwaggerEndpoint("/openapi/v1.json", "Teledoc Api Services");
options.RoutePrefix = "swagger";
```

## Как скачать и запустить проект

### 1. Клонировать репозиторий

```bash
git clone https://github.com/Archikey/Teledoc.Api.git
```

### 2. Перейти в папку проекта

```bash
cd Teledoc.Api
```

### 3. Восстановить зависимости

```bash
dotnet restore
```

### 4. Запустить Docker Desktop

Перед запуском проекта необходимо убедиться, что Docker Desktop запущен.

Docker Desktop используется для поднятия PostgreSQL в контейнере.

### 5. Запустить проект через AppHost

Проект запускается через Aspire AppHost.

```bash
dotnet run --project TeledocTestApp
```

После запуска AppHost поднимет необходимые сервисы:

- API
- PostgreSQL
- Redis
- Aspire Dashboard

### 6. Открыть Swagger

Swagger доступен по адресу:

```text
https://server-main-teledoctestapp.dev.localhost:7209/swagger
```

### 7. Запустить тесты

```bash
dotnet test
```

## Запуск через Visual Studio 2026

Открыть файл решения:

```text
TeledocTest.slnx
```

В качестве стартового проекта выбрать:

```text
TeledocTestApp
```

Запустить проект.

Перед запуском должен быть запущен Docker Desktop.
